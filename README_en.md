# 📱 Mobile Application Testing (Android): Urban.Lunch
![Project](https://img.shields.io/badge/Project-Mobile_Testing-blue) ![OS](https://img.shields.io/badge/OS-Android-green) ![Tool](https://img.shields.io/badge/Tool-Android_Studio-orange) ![Tracking](https://img.shields.io/badge/Tracking-Jira-blue)

#### 📌 Project Summary (STAR Methodology)
*   **Situation:** The development team launched the first Android version of **Urban.Lunch**, a mobile application for ordering food from various restaurants and delivering it to pickup points. It was essential to guarantee usability, shopping cart counter logic, and proper device permission management (such as geolocation) prior to its launch.
*   **Task:** Design and execute comprehensive checklists by testing the application directly in a mobile environment (Android Studio emulator / Android device) and report any UI/UX or functional anomalies in Jira.
*   **Action:**
    *   Executed functional tests by navigating the pickup points map, dish detail screens, and order timer tracking.
    *   Stressed the user interface by checking for text overlaps and dynamic state changes (e.g., selection colors on dishes and buttons).
    *   Validated hardware/software integration by denying location access to verify exception handling (Error notifications).
*   **Result:** Isolated more than 10 defects, including critical failures where the unit counter did not increase when pressing the '+' button, restaurant name overlaps in the interface, and permission management vulnerabilities (the app failed to display an error when geolocation was denied).
